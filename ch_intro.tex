\chapter{引言}

\section{研究背景}

随着边缘智能、物联网和计算机视觉技术不断发展，视频监控系统已经不再只是传统意义上的被动录像设备，而是逐步向具备实时感知、边缘分析和主动控制能力的智能终端转变。在早期视频监控应用中，摄像机主要负责采集和保存画面，后续分析通常依赖人工查看或中心平台回放。随着应用场景变得更加复杂，现场响应的实时性要求也在提高，监控设备不仅要“看得见”，还需要在本地完成目标识别、事件判断和运动控制，从而把从感知到执行的链路尽量缩短。

布控球是一类便携式移动监控设备，通常集成摄像头、云台、通信模块和供电模块，能够比较快速地部署到临时布防、应急救援、施工巡检、交通保障和偏远区域监测等现场。与固定摄像机相比，布控球有部署灵活、覆盖范围可调、远程操控能力较强等特点，因此在智慧城市和行业安防系统中具有较高的工程价值。在实际使用过程中，布控球可能会被安装在三脚架、车辆、杆体或临时支架上，设备姿态和监控视角会随着部署条件变化而变化，这也使得自动化控制能力会直接影响现场使用效率。

但是，传统布控球的云台控制方式仍然以人工遥控、预置点切换或固定轨迹巡航为主。当画面中出现快速移动目标、多人同时进入、目标短时遮挡或光照突变时，人工操作往往很难及时把目标保持在画面中心，固定巡航策略也无法根据目标位置动态调整云台姿态。如果把视频流全部上传到云端处理，系统又会受到网络带宽、传输时延和隐私安全等因素限制。在应急处置和移动巡检场景下，网络条件并不总是稳定，现场人员也难以长期保持高强度监控，因此仅依赖人工和云端的方案，难以同时满足低延迟、低成本和高可用性的要求。

因此，把轻量级视觉检测算法和自适应云台控制策略部署在边缘端，使布控球能够在本地完成目标检测、目标选择和控制决策，是提升移动监控设备智能化水平的一个重要方向。边缘端方案的价值在于把“发现目标”和“调整视角”两个环节尽量放在设备附近完成。一方面，系统能够减少视频全量上传带来的带宽压力；另一方面，控制指令可以根据最新画面更快生成，有利于提高目标居中和持续跟踪的稳定性。

本课题面向海康威视布控球设备，围绕“视频采集--目标检测--目标记忆--云台控制--人机交互”这一闭环链路，研究并实现一种自适应云台控制嵌入式设备原型。系统以 Python 为主要开发语言，集成 HCNetSDK、PlayCtrl、YOLOv8n、OpenCV、Tkinter 和轻量级人体 ReID 模块，实现实时视频流获取、人体目标检测、目标编号与选择、PTZ 自动跟踪、倒挂模式适配以及手动控制等功能。与单纯的离线检测算法验证不同，本课题更强调真实设备接入和系统闭环运行，重点处理 SDK 调用、视频流解码、检测线程调度、云台控制节奏和界面交互之间的协同问题。

从应用场景看，布控球智能跟踪系统的难点并不只在于“能否检测到人”，还在于检测结果能否稳定地转化为设备动作。目标检测模型通常只输出边界框和类别置信度，而云台设备需要的是方向、速度和持续时间等控制信息。如果中间缺少目标选择和控制策略，检测框的轻微变化就可能被放大为云台抖动。因此，本文在系统设计中将人体检测、目标记忆和 PTZ 控制作为一个整体考虑，而不是把它们看作互不相关的独立模块。

\section{研究意义}

本研究具有一定的工程应用意义。一方面，系统把目标检测与 PTZ 控制部署在边缘端，减少了对云端实时分析链路的依赖，有利于降低网络传输压力，并提高控制响应速度。另一方面，系统面向实际布控球设备完成 SDK 调用、码流解码、实时帧获取和云台控制接口适配，能够为同类设备的二次开发提供一定参考。本文还在简单最大目标跟踪的基础上加入人体 ReID 记忆和指定 ID 跟踪机制，使系统在多人场景下具有更好的目标连续性，减少频繁切换跟踪对象的问题。同时，系统提供可视化控制界面和倒挂模式切换功能，使设备更贴近实际部署和调试需求。

从项目实现过程看，本研究也具有一定的综合训练价值。课题涉及设备通信、接口适配、实时图像处理、深度学习推理、目标关联、反馈控制和人机交互设计等内容。单独看每一个模块，难度并不完全体现在算法本身；但当这些模块被放入同一个实时系统后，就需要同时考虑数据格式、线程节奏、异常恢复和资源释放等工程问题。因此，该课题能够较好体现人工智能算法从模型调用走向设备应用时需要经历的系统集成过程。

从学术和技术融合角度看，本课题把轻量级目标检测、视觉伺服控制、嵌入式实时处理和视频监控设备接口适配结合起来。它既关注算法在边缘环境下的实时性，也关注云台控制的稳定性和人机交互的可用性，对计算机视觉算法的工程化落地具有一定参考价值。本文并不追求提出新的通用检测网络，而是围绕布控球这一具体设备，探索如何把成熟模型、轻量目标记忆和规则控制策略组合成一个可以运行的应用系统。

对于本科毕业设计而言，本课题也能够较好体现“理论方法”和“工程实现”之间的联系。目标检测、目标跟踪和反馈控制在理论上都有较多成熟方法，但落到真实设备上时，还需要考虑 SDK 接口、图像格式、线程调度、异常处理和人工调试等问题。通过完成一个可运行的原型系统，可以更直观地理解人工智能算法从论文模型走向实际设备时需要经历的工程化过程。

\section{研究目标}

结合任务书、开题报告和中期检查材料，本文的研究目标包括以下几个方面：

\begin{enumerate}
    \item 完成布控球硬件接口适配。基于海康威视 HCNetSDK 与 PlayCtrl 播放库，实现设备初始化、登录认证、实时预览、码流解码和 PTZ 控制等功能，构建稳定的视频采集与控制通道。
    \item 实现轻量级人体目标检测模块。选用 YOLOv8n 作为检测模型，完成实时帧读取、人体类别筛选、置信度过滤、目标坐标输出和检测结果可视化，为控制算法提供输入。
    \item 设计自适应云台控制策略。根据目标中心与画面中心偏差生成 PTZ 控制指令，并结合阈值死区、比例脉冲、指令冷却和指数平滑机制降低跟踪抖动。
    \item 增强多目标场景下的目标连续性。构建轻量级人体 ReID 记忆模块，为人体目标分配稳定 ID，支持自动目标选择和指定 ID 跟踪。
    \item 完成系统集成与测试。将视频采集、目标检测、目标记忆、云台控制和图形界面整合为完整原型，并从实时性、稳定性和可用性等方面进行验证。
\end{enumerate}

上述目标之间具有比较清晰的递进关系。硬件接口适配是系统运行的基础，只有稳定获得视频帧并能够下发云台指令，检测和控制才有实际意义；目标检测模块提供目标位置，是闭环控制的数据源；人体 ReID 记忆模块用于改善多目标场景下的目标一致性；PTZ 控制策略把图像坐标偏差转换为可执行的机械动作；图形界面和测试验证则用于保证系统能够被用户观察、调试和复现。

\section{主要研究内容}

围绕上述目标，本文主要从以下几个方面开展研究：

\subsection{硬件接口适配与视频采集}

系统通过 HCNetSDK 完成布控球设备的初始化、登录和云台控制，通过 PlayCtrl 完成实时码流解码，并把解码后的图像帧缓存为检测模块能够读取的数据。这一部分主要解决 SDK 接口适配、设备登录失败处理、预览句柄管理、解码回调保存和资源释放等工程问题。由于 SDK 回调数据并不是检测模型可以直接处理的图像格式，系统需要在解码回调、图像保存、帧读取和尺寸归一化之间建立稳定的数据通路，同时要保证检测线程读取图像时不会阻塞预览链路。

\subsection{轻量级目标检测与坐标输出}

检测模块选用 YOLOv8n 预训练模型，对布控球画面中的人体目标进行实时检测。系统设置输入尺寸、置信度阈值和半精度推理选项，用来在实时性与检测质量之间取得平衡。检测结果经过类别筛选和坐标转换后，会输出边界框、中心点、置信度和面积等信息，为目标选择和 PTZ 控制提供结构化数据。考虑到布控球画面中可能存在视角变化、目标远近变化和背景干扰，系统没有直接使用全部检测类别，而是重点关注人体类别，并把检测框面积作为自动目标选择的重要依据。

\subsection{人体 ReID 记忆与目标选择}

针对多目标场景中“最大目标策略”容易发生目标切换的问题，本文设计了轻量级人体 ReID 记忆模块。该模块利用人体框位置比例和平均颜色等低成本特征计算相似度，使目标在短时缺失后重新出现时，仍有机会被合并为同一 ID。系统支持自动跟踪最大人体，也支持用户在界面中循环选择指定 ID。这样既保留了自动模式下快速选择主目标的便利性，也允许用户在多人场景中明确指定需要跟踪的对象，从而提高系统的可控性。

\subsection{自适应 PTZ 控制}

控制模块根据目标中心点与画面中心点的偏差确定水平和垂直转动方向。为了避免微小检测抖动引发云台频繁动作，系统设置了水平和垂直死区阈值；为了兼顾远距离偏差的快速回正与近中心区域的细致调整，系统会根据偏差大小调整单次 PTZ 脉冲时长，并通过冷却间隔限制控制指令频率。该策略采用不依赖复杂动力学建模的工程化控制方式，参数比较直观，调试成本较低，适合在原型系统和真实设备测试中逐步整定。

\subsection{可视化界面与系统集成}

系统构建了人机交互模块，提供视频预览、目标状态呈现、跟踪模式切换、倒挂适配和人工接管等功能。交互层与后台检测线程、PTZ 控制线程协同工作，使测试人员能够看到目标框、目标 ID、偏移量和云台运动效果。该模块的设计重点不是复杂交互形式，而是支撑系统状态管理、控制方向验证以及异常场景下的人工干预。

\section{论文结构}

本文共分为六章。第一章介绍课题背景、研究意义、研究目标和主要内容。第二章综述布控球云台控制、目标检测、目标跟踪与边缘智能相关技术。第三章分析系统需求并给出总体架构设计。第四章详细说明视频采集、目标检测、人体 ReID、PTZ 控制和界面集成等关键模块的实现。第五章给出系统测试方案、初步测试结果和性能分析。第六章总结全文工作，并讨论模型优化、复杂场景适应性和云边协同等工程拓展方向。
